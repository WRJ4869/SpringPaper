# 🌸 SpringPaper（春笺）

**AI 提高效率，判断仍属于老师。**

夜深人静时，面对屏幕上数百份略显稚嫩的笔迹，每一次打分，其实都是一次跨越纸面的对话。

SpringPaper（春笺）诞生于真实的教育前线。它不是一台冰冷傲慢、妄图替代教师的自动阅卷机器，而是一位极其安静、克制、可靠的书桌陪伴者。它负责接管那些极其消耗精力的重复劳动：精准截图、视觉识别、鼠标点击与分数录入；而将最极其宝贵的精力，留给老师去感知文字背后的真实温度。

算力负责奔跑，但最终的裁量与判断，始终属于老师。

## 核心功能

- AI 作文建议评分：基于作文截图、题目和评分标准生成建议分、档位、优点、问题与备注。
- AI 输出模式：支持极速、标准、复核三档输出；单篇试评和自动连评可分别选择，默认推荐为单篇标准、连评极速。
- 智学网页面截图：通过校准后的作文区域截取当前卷面。
- 半自动提交：老师确认建议分后，可用“按钮打分”或“输入框打分”两种方式提交。
- 自动连评：按设定数量连续处理试卷，遇到低置信或疑问卷可自动暂停。
- 人工复核机制：所有建议分都保留人工确认入口，老师可以随时停止、修正、继续。
- 校准区域：作文区域、分数网格、分数输入框、提交按钮坐标可在本地保存。
- 本地轻量日志：界面只显示最近记录，本地日志按天保存并自动清理。
- 温暖工作台 UI：基于 CustomTkinter，遵循 SpringPaper Design System。

## 使用前准备

1. 安装 Python 3.10 或更高版本。
2. 安装依赖：

```powershell
pip install -r requirements.txt
```

3. 准备可用的视觉模型 API Key，例如硅基流动、OpenAI 兼容接口等。
4. 在系统环境变量中设置 API Key，或在软件界面中临时输入。
5. 打开智学网阅卷页面，并进入作文题阅卷界面。

## 启动方式

Windows 下可双击：

```text
启动春笺_SpringPaper.bat
```

也可以在项目根目录运行：

```powershell
powershell -ExecutionPolicy Bypass -File .\run_springpaper.ps1
```

或直接运行：

```powershell
python .\src\springpaper.py
```


面向非开发者时，只需要把 `SpringPaper-v1.0.0.zip` 发给对方。解压后双击 `SpringPaper.exe`，再按 `首次使用说明.md` 完成 API、评分标准和区域校准即可。

## 基本使用流程

1. 在“AI与标准”中选择模型，填写 Base URL、API Key、题目和评分标准。
2. 在“AI与标准”中确认 AI 输出模式：单篇试评和自动连评都可在“极速 / 标准 / 复核”之间切换。
3. 在“校准”中校准作文区域，并按提交方式校准分数网格或分数输入框，再记录网页提交按钮。
4. 回到“阅卷”，点击“截图并评分”获取 AI 建议。
5. 人工确认建议分，如认可则点击“采纳并提交”；默认使用按钮打分，也可在“阅卷”中切换为输入框打分。
6. 稳定后可使用“自动连评”，遇到疑问卷时停止并人工复核。
7. 在“记录”中查看最近评分过程、复制可见记录或继续/停止连评。

## AI 输出模式

- 极速：只返回 `score`、`confidence`、`recheck`，适合自动连评，目标是减少输出 Token 和等待时间。
- 标准：返回 `score`、`confidence`、`recheck` 和一句 30 字以内的理由，适合单篇试评。
- 复核：返回完整结构化 JSON，适合疑难卷或人工复查时查看更多判断依据。

单篇试评和自动连评只是不同使用场景，不限制模式选择：单篇可以用极速，连评也可以用标准或复核。

## 安全与隐私

- API Key 不应写入代码，也不应提交到 GitHub。请使用环境变量或本地输入。
- `config.json`、`logs/`、`last_essay_capture.png` 默认不应上传，它们可能包含本地配置、阅卷记录或试卷截图。
- 使用前请确认符合学校、平台和数据安全要求。
- AI 输出只作为建议，最终评分责任与判断权始终属于老师。

## 当前版本限制

- 当前主要适配智学网网页阅卷界面，页面结构变化后可能需要重新校准。
- 视觉模型对手写字识别和作文质量判断存在波动，尤其在字迹潦草、卷面遮挡、截图过小等情况下。
- 自动提交依赖屏幕坐标，浏览器缩放、窗口布局、分数按钮或输入框位置变化后需要重新校准。
- 本项目尚未提供正式安装包；当前以源码运行和后续 PyInstaller 打包为主。
