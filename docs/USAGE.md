# 智学网作文 AI 半自动阅卷助手

## 启动

最简单：双击 `启动阅卷助手.bat`。

右键 `run_assistant.ps1`，选择“使用 PowerShell 运行”。

如果系统阻止脚本运行，可以在 PowerShell 中进入本目录后执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\run_assistant.ps1
```

## 第一次使用

1. 打开智学网作文阅卷页面。
2. 启动助手。
3. 选择 API 供应商，填入 API Key，或提前设置对应环境变量。
4. 点击“测试API连接”，确认 key、base URL 和模型可用。
5. 点击“校准作文区域”，拖选左侧作文正文区域。
6. 点击“测试视觉输入”，确认模型能接收截图，并能摘录作文中的可见文字。
7. 添加作文题目、参考答案、评分标准图片，或在补充说明框里粘贴文字标准。
8. 点击“校准分数网格”，拖选右侧 0-50 分按钮区域。
9. 点击“记录网页上的‘提交分数’按钮”，3 秒内把鼠标移到智学网网页右侧的“提交分数”按钮中心，等待自动记录。
10. 点击“截图并评分”。
11. 确认建议分后，点击“采纳并提交”。

## 人工卷面判断

模型容易把“内容完整”看得过重，而低估卷面、涂改和字迹对作文定档的影响。正式使用时，建议你先扫一眼卷面，在“人工卷面判断”里选择：

- `自动`：不额外修正，完全采用 AI 建议分。
- `整洁`：不调整。
- `较整洁`：最高不超过 44。
- `欠整洁`：最高不超过 40。
- `凌乱难辨`：最高不超过 36。

点击“采纳建议分并提交”时，工具会使用修正后的分数。

## 模型对比

如果模型连续把普通作文或卷面较差的作文打到 42 左右，可以先换模型测试同一篇作文。工具里有“快速选择模型”下拉：

- `Qwen/Qwen3-VL-32B-Instruct`：当前默认，优先测试它是否比 30B-A3B 更会看卷面和区分档位。
- `Qwen/Qwen3-VL-30B-A3B-Instruct`：速度较快，之前能稳定出分，但容易把普通完整作文打到 42。
- `Qwen/Qwen3-VL-8B-Instruct`：更快，适合跑流程，不建议作为最终尺度依据。
- `Qwen/...-Thinking`：可以试一两篇看判断是否更细，但可能更慢，也更容易输出不稳定。

建议对同一篇作文分别测试 `32B-Instruct` 和 `30B-A3B-Instruct`。如果两个模型都忽略卷面，就启用“人工卷面判断”作为最终闸门。

## 进度怎么看

点击“截图并评分”后，下方日志会依次显示：

- 正在截图作文区域
- 作文截图完成
- 已加入材料图片
- 正在发送请求给模型
- 模型已返回，正在解析评分结果
- AI 建议分和耗时

如果长时间停在“正在发送请求给模型”，通常是视觉模型响应慢、模型排队、网络慢，或图片太大。

视觉测试通过的标志：日志里出现 `visible_text_samples`，并且里面确实是作文图片中的词句。只返回 `vision_ok: true` 还不够。

## 题目与评分标准

正式批作文前，建议至少提供一种材料：

- 作文题目截图
- 学校/年级作文评分标准截图
- 参考答案或评分细则截图
- 在补充说明框中粘贴文字版评分标准

如果不提供，工具会使用通用初中作文 50 分标准，只能作为粗略初评。

## 可粘贴的本次考试补充说明

这类临时尺度建议写在工具里的“作文题目/评分标准补充说明”框中，不建议写进全局评分提示词。

```text
第一张材料是作文题目和写作要求；第二张材料是作文评分标准。请严格按照第二张评分标准给出 0-50 分整数建议分。题目为“留一点____给自己”，要求结合亲身体验，写出真情实感，不少于600字。

这是期末考试作文阅卷，请按照评分标准改。符合题意、中心明确、叙事完整、语言通顺、有真实体验的作文，应在对应档位内给到合理分数；特别优秀的作文可以给满分或接近满分；其余正常完成、没有明显硬伤的作文，可在不突破分档标准的前提下适当宽一点。

评价可以具体，但不要冗长。优点和问题尽量指向作文中的真实表现，例如中心、材料、细节、结构、语言、结尾升华、字数和卷面。不要仅凭版面感觉判断字数不足；只有明显少于要求或能较确定低于600字时，才因字数扣分。
```

## API 供应商

### OpenAI

- 供应商：`openai`
- Base URL：留空
- 默认模型：`gpt-4o-mini`
- 环境变量：`OPENAI_API_KEY`

### 硅基流动

- 供应商：`siliconflow`
- Base URL：`https://api.siliconflow.cn/v1`
- 默认模型：`Qwen/Qwen3-VL-30B-A3B-Instruct`
- 环境变量：`SILICONFLOW_API_KEY`

硅基流动要选择支持视觉输入的模型。若模型名不可用，可以在“模型”输入框里换成你账户后台可用的视觉模型。

如果测试 API 时出现 `Model disabled`，说明当前模型在你的账号或平台上不可用。请点击“列出当前账号可用模型”，从日志里选择带 `VL`、`vision`、`glm-4v`、`OCR` 等字样的模型名，复制到“模型”输入框后再测。

从你账号列出的模型看，优先推荐：

- `Qwen/Qwen3-VL-30B-A3B-Instruct`：先用这个，速度和效果比较均衡。
- `Qwen/Qwen3-VL-8B-Instruct`：更快，但作文细读可能弱一些。
- `Qwen/Qwen3-VL-32B-Instruct`：可能更稳，但通常更慢。
- `deepseek-ai/DeepSeek-OCR`：更像 OCR 模型，适合识别文字，不建议直接承担作文评分。

不建议第一轮用 `Thinking` 模型，速度会更慢，也更容易输出非 JSON 的长解释。

### DeepSeek

- 供应商：`deepseek`
- Base URL：`https://api.deepseek.com`
- 默认模型：`deepseek-v4-flash`
- 环境变量：`DEEPSEEK_API_KEY`

注意：当前工具是“看作文截图评分”。DeepSeek 普通聊天模型偏文本，若接口不支持图片输入，会评分失败。它更适合后续接 OCR：先把作文识别成文字，再交给 DeepSeek 评分。

### 自定义 OpenAI 兼容接口

- 供应商：`custom`
- Base URL：填写服务商的 `/v1` 或兼容地址
- 模型：填写服务商提供的视觉模型名
- 要求：支持 `/chat/completions` 和 `image_url` base64 图片输入。

## 建议

- 第一次先取消“确认后自动提交”，用 5-10 篇人工校准尺度。
- 正式考试不要全自动无人值守，至少保留人工确认和抽检。
- 不要截图姓名、考号等学生个人信息。
- 硅基流动 72B 视觉模型可能比较慢；若后台有更小的视觉模型，可以在“模型”框中替换后先测试视觉输入。
