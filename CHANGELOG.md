# Changelog

## v1.1.0 - 2026-07-04

SpringPaper 增加“提交方式”适配层，让不同智学网页面都能更稳地落分。

### Added

- 新增“按钮打分 / 输入框打分”两种提交模式。
- 新增“校准分数输入框”入口，支持记录分数输入框中心坐标。
- 输入框打分模式下，采纳建议分后会点击输入框、全选清空、输入分数，再点击已记录的提交按钮。
- 自动连评启动前会根据当前提交方式检查必要校准项，避免模式不匹配时误操作。

### Notes

- 默认仍为“按钮打分”，现有 0–50 分数字按钮流程不受影响。
- 如果输入框提交失败，可直接切回“按钮打分”，重新使用原分数网格校准。

## v1.0.0 - 2026-07-04

SpringPaper（春笺）第一次进入 Release-ready 阶段。

### Added

- AI 作文建议评分：支持基于作文截图、题目和评分标准生成建议分与简要评语。
- 智学网页面截图：支持校准作文区域后自动截取当前卷面。
- 半自动提交：支持人工确认建议分后自动点击分数并提交。
- 自动连评：支持按设定数量连续阅卷，遇到疑问或低置信结果时暂停。
- 人工复核：保留老师确认、修正、停止和继续的完整工作流。
- 校准区域：支持作文区域、分数网格和提交按钮坐标记录。
- 本地轻量日志：界面只显示近期记录，本地日志按天保存并自动清理。
- CustomTkinter UI：完成从原生 Tkinter 到温暖桌面工作台界面的升级。
- SpringPaper 品牌命名：确立“春笺 / SpringPaper”名称与“AI 提高效率，判断仍属于老师”的产品理念。
- About 弹窗：加入极简版本信息与产品理念展示。
- 发布包封装：加入 `SpringPaper.exe` 打包脚本、首次使用说明、配置模板、图标与版本信息。

### Notes

- 当前版本面向本地使用与源码运行，后续将继续补充打包产物与正式 Release 文件。
- API Key、阅卷日志、截图和本地配置默认不应上传到 GitHub。
