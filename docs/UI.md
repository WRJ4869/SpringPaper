# SpringPaper UI Guide

## Layout

SpringPaper 是工作台，不是官网。

启动后的第一屏应尽快出现：

1. 产品名与一句极短理念。
2. 标签页。
3. 当前工作区。

随机文案、状态提示、版本信息应保持轻量，不抢占阅卷区域。

## Color Semantics

颜色表达语义，而不是单纯装饰。

- Primary：樱花粉，用于“截图并评分”等主要动作。
- Success：鼠尾草绿，用于“采纳并提交”等确认动作。
- Auto：薰衣草紫，用于“开始/继续自动连评”。
- Danger：珊瑚红，用于“停止自动连评”等风险动作。
- Secondary：灰白，用于复制、清空、测试、校准等普通动作。

强调色控制在界面可见面积的 10% 左右。

## Typography

马卡龙按钮不使用粗黑大字。

按钮文字应使用常规字重和柔和深灰，确保清晰但不刺眼。

标题可以加粗，但不应压迫工作区。

## Cards

避免深层卡片嵌套。

推荐：

```text
Tab
└── Card
    └── Controls
```

避免：

```text
Card
└── Card
    └── Scroll
        └── Card
```

## Compact Window Rules

窗口变小时：

1. 优先隐藏说明文字。
2. 保留关键按钮。
3. 保留滚动能力。
4. 状态提示移到底部轻量显示。

不可让“最多连评”“提交等待”“复制记录”“清空界面”等关键操作被裁切到无法访问。
