# SpringPaper Roadmap

## v1.0.0

首个 Release-ready 版本。

- AI 作文建议评分。
- 智学网页面截图。
- 半自动提交。
- 自动连评。
- 人工复核。
- 区域校准。
- 本地轻量日志。
- CustomTkinter UI。
- SpringPaper 品牌命名与 About 弹窗。

## v1.0.x

小修小补，不新增大功能。

- 修复滚动、缩放、按钮遮挡等可用性问题。
- 优化日志可读性。
- 调整文案与 README。
- 补充使用手册和截图。

## v1.1.0

提升不同智学网页面下的提交适配能力。

- 保留按钮打分模式，继续适配 0-50 分数字按钮。
- 新增输入框打分模式：点击输入框、清空、输入分数、再提交。
- 新增分数输入框坐标校准。
- 默认仍使用按钮打分，避免破坏现有阅卷流程。

## v1.1.x

继续打磨稳定性与可观察性。

- 增加更清晰的运行状态。
- 增加模型耗时、置信度、暂停原因统计。
- 优化疑问卷人工处理流程。

## v1.2.0

优化 AI 输出策略，减少自动连评等待时间。

- 新增极速、标准、复核三档输出模式。
- 自动连评默认极速，只保留分数、置信度和复核标记。
- 单篇试评默认标准，保留一句简短理由。
- 单篇试评和自动连评都可在三档之间独立切换。
- 复核模式保留完整 JSON，供疑难卷查看。
- Prompt 采用 Base Prompt + Mode Prompt 组合，避免复制三份提示词。

## v1.3.0

探索“智能输出模式”。

- 根据置信度、复核标记和异常信号自动决定输出详细程度。
- 高置信稳定卷走极速。
- 中等置信卷走标准。
- 低置信、跑题、字数不足、疑似异常卷走复核。
- 目标是减少老师管理 AI 的负担，让 SpringPaper 自己判断什么时候该快、什么时候该认真。

## v1.4.0

扩展使用场景。

- 支持更多题型或学科的评分模板。
- 支持多套评分标准管理。
- 支持更方便的模型切换和模型对比。

## Future

- Focus Mode：长时间阅卷时隐藏非必要信息。
- PyInstaller 正式打包。
- GitHub Release 自动化。
- 更完整的教师使用手册。
